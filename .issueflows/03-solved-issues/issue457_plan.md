# Plan — issue #457 (Stage 1.12)

*Keys live in columns, never in an index* (polars plan decision 3) applied in
place, v1.x-safe:

1. **A1 raw:** `arbin_res._post_process` stops promoting `data_point` to the
   raw index. The frozen v8 storage keeps its stored index (the write-side
   promotion in `cellpy_file/write.py` and the HDF5 where-clauses are storage
   boundary concerns), and the read path de-indexes right after `select` —
   raw comes back with a RangeIndex and the `data_point` column.
2. **A2 summary:** the hardcoded `cycle_index_as_index = True` locals in both
   summary builders flip to False; `split`/`with_cycles` stop re-promoting;
   the read path restores `cycle_index` as a column when loading old files
   (stored index was set with drop=True, so `reset_index(drop=False)`).
3. **A3 journal:** the four `pages.set_index(filename)` sites gain
   `drop=False` — the key now also lives in a column; removing the index
   itself rides utils wave 1.
4. **Warn-only index lint:** `tests/test_index_lint.py` scans the package for
   index idioms outside the sanctioned boundary modules
   (`readers/cellpy_file/`, `parameters/legacy/`) and warns with a count —
   never fails; polars plan Phase D flips it to a hard ban.
5. **Deliberate characterization updates** (documented in each test):
   index-name assertions inverted, golden lookups moved from `.loc[cycle]` to
   column filters, journal missing-header count 1 → 0,
   `remove_outliers_from_summary_on_index` made column-aware (still accepts
   cycle numbers), pipeline_smoke goldens regenerated by script (summary
   legitimately gains the `cycle_index` column).
